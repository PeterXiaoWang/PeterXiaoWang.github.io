Surface 与 BufferQueue

Surface 获取一个画布对象供上层 View 进行绘制，而绘制的数据需要有一块内存来存放，而这块区域是从 BufferQueue 中获取的。在 View 绘制完成后，再将这块内存添加到 BufferQueue 中，供 SurfaceFlinger 进行消费。

unlockCanvasAndPost（提交 Canvas） 方法主要的逻辑是将绘制好的 Canvas 中的 Buffer 更新到 BufferQueue 中供 SurfaceFlinger 消费。

BufferQueue 与 SurfaceFlinger

SurfaceFlinger 从 BufferQueue 里获取图像数据进行处理后交给图形显示设备进行展示。

图像生产者

  1. Skia：Skia 是一个 2D 图形库，在 Android 系统中扮演着重要的角色，它被用于处理 UI 界面的绘制、图形渲染、文本渲染、图像处理等方面。Android 系统中的许多 UI 元素、控件和视图都是通过 Skia 库来实现绘制和渲染的。Skia 库的特点包括高效的图形渲染、跨平台支持、灵活的绘图接口、优化的文本渲染等。在 Android 开发中，开发者可以通过 Android 的绘图API（如Canvas、Paint等）来利用 Skia 库进行图形绘制和处理，从而实现各种复杂的UI效果和交互。
  2. OpenGL ES：OpenGL ES（OpenGL for Embedded Systems）是一种用于嵌入式系统的图形库，用于实现 2D 和 3D 图形渲染。OpenGL ES 是 OpenGL 图形标准的一个子集，专门针对移动设备、嵌入式系统和游戏开发进行了优化和精简。OpenGL ES 提供了一套跨平台的 API，允许开发者在 Android 设备上进行图形渲染和处理。通过使用 OpenGL ES，开发者可以实现各种高质量的图形效果、动画和交互，为应用程序提供更加生动和吸引人的用户界面。在 Android 开发中，开发者可以使用 OpenGL ES 来进行图形渲染、纹理映射、光照效果、投影变换等操作，从而实现各种复杂的 3D 场景和动态效果。OpenGL ES 提供了丰富的图形函数和接口，开发者可以利用这些接口来创建各种图形效果和交互体验。OpenGL ES 在 Android 系统中被广泛应用于游戏开发、图形应用程序、虚拟现实（VR）应用程序等领域。虽然相对于 Vulkan 等新兴的图形API，OpenGL ES 在性能和灵活性上可能有一定限制，但它仍然是一个强大且成熟的图形库，适用于许多类型的应用程序开发。

Vulkan：Vulkan 是一种跨平台的低级图形 API，用于实现高性能的 3D 图形渲染。Vulkan API 相对于传统的图形 API（如OpenGL ES）具有更低的驱动开销、更好的多线程支持、更细粒度的控制、更高的性能和更低的 CPU 开销等优势。通过使用 Vulkan API，开发者可以更好地利用设备的硬件资源，实现更复杂、更高性能的图形渲染和计算任务。在 Android 系统中，Vulkan API 被广泛应用于游戏开发、图形应用程序和虚拟现实（VR）应用程序等领域。通过使用 Vulkan API，开发者可以实现更高质量的图形效果、更流畅的动画和更快速的渲染速度，从而提升用户体验。

图像消费者

SurfaceFlinger 是 Android 中最重要的一个图像消费者，其具有以下特性：

  • 负责消费生产者生产的 Surface 并使用 WindowManager 提供的信息，将 Surface 合成到屏幕上。
  • 和 ActivityManagerService 一样，是一个系统服务。
  • SurfaceFlinger 会使用 OpenGL 和 HWC 来完成图层合成。

图像缓冲区

Android的图形缓冲区主要由 4 个部分组成：

  • GraphicBuffer 是真正被分配内存，并能存储图形数据的缓冲区。
  • Surface 是提供给图像生产者控制缓冲区的类，Surface 持有 GraphicBufferProducer (下文简称 GBP)，GBP 用来创建或获取可用的 GraphicBuffer 以及提交完成绘制后的 GraphicBuffer。
  • Layer 是提供给图像消费者获取缓冲区的类，Layer 持有 GraphicBufferConsumer (下文简称 GBC)，GBC 用来获取和释放 GraphicBuffer。
  • BufferQueue 是 Android 图形组件之间的粘合剂，能够将生产者与消费者结合到一起 (通过 Surface 和 Layer)。其中 Surface 持有 BufferQueue 中的 GBP，而 Layer 持有 BufferQueue 中的 GBC，GBP 和 GBC 的 GraphicBuffer 都存储在 GraphicBufferCore 的 slots 数组中。

在 Android 平台上创建的每个窗口都由 Surface 提供支持的，一个 Surface 对应了一块缓冲区。Android 屏幕上的图像其实是由多个 Surface 合成而来，图像消费者 SurfaceFlinger 就是负责合成的这项工作。SurfaceFlinger 操作的对象是 Layer，一个 Surface 会对应于 SurfaceFlinger 上的一个 Layer，Layer 本质上是 Surface 和 SurfaceControl 的组合，Surface 充当着在生产者和消费者之间传递数据的缓冲区，而 SurfaceControl 是 Surface 的控制类。

总结

从较底层的角度来看一次生产和消费流程：

  • 图像生产者 (Skia、OpenGL ES、Vulkan) 获取一块内存 (通过 Surface 调用 dequeue 函数获取一块缓冲区 GraphicBuffer) 进行绘制，绘制的图像会被存储在缓冲区上，绘制完成后将结果提交到 BufferQueue (通过 Surface 调用 queue 函数，将 GraphicBuffer 提交给 BufferQueue)；
  • 当图像消费者 SurfaceFlinger 收到 VSync 信号时 (SurfaceFlinger 会监听 VSync 信号)，获取一块已完成绘制的内存 (通过 Layer 调用 acquire 函数获取已绘制完成的 GraphicBuffer) 进行处理，在处理完成后将数据送至显存，并做一些收尾工作 (通过 Layer 调用 release 函数释放已处理完成的 GraphicBuffer)。
